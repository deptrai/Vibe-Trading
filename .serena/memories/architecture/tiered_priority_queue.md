Decided to implement Tiered Priority Queue (Story 5.1) using JWT-based routing. Nowing sends a JWT token in the Authorization header. `api_server.py` extracts `user_tier` using `pyjwt` with the `JWT_SECRET` key. If the token is invalid/missing, it safely falls back to `standard` tier. Celery uses dynamic queues like `backtest.premium`, `backtest.standard`, `rl_optimization.premium`, and `rl_optimization.standard`. The 10:1 token-bucket fairness ratio is effectively simulated by listing premium queues before standard queues in `start_worker.sh` combined with Celery's autoscale settings (`--autoscale=10,3`).
